### 1. 跳跃游戏

给定一个非负整数数组，你最初位于数组的第一个位置。

数组中的每个元素代表你在该位置可以跳跃的最大长度。

判断你是否能够到达最后一个位置。

示例 1:

	输入: [2,3,1,1,4]
	输出: true
解释: 我们可以先跳 1 步，从位置 0 到达 位置 1, 然后再从位置 1 跳 3 步到达最后一个位置。
示例 2:

	输入: [3,2,1,0,4]
	输出: false
解释: 无论怎样，你总会到达索引为 3 的位置。但该位置的最大跳跃长度是 0 ， 所以你永远不可能到达最后一个位置。

### 解题思路

遍历数组中所有的点，记录每一个点，能跳到的最大位置

如果这个最大位置能够到达最后一个位置，返回true， 否则false

可以把  i <= rightMax 去掉，更方便理解。

时间复杂度：O(n)，其中 n 为数组的大小。只需要访问 nums 数组一遍，共 n 个位置。

空间复杂度：O(1)，不需要额外的空间开销。


如果某一个作为 起跳点 的格子可以跳跃的距离是 3，那么表示后面 3 个格子都可以作为 起跳点。

可以对每一个能作为 起跳点 的格子都尝试跳一次，把 能跳到最远的距离 不断更新。

如果可以一直跳到最后，就成功了。



### 解题思路

看到一个思路说：[3,2,1,0,4]

1. 如果某一个作为 起跳点 的格子可以跳跃的距离是 3，那么表示后面 3 个格子都可以作为 起跳点。
2. 可以对每一个能作为 起跳点 的格子都尝试跳一次，把 能跳到最远的距离 不断更新。
3. 如果可以一直跳到最后，就成功了。

时间复杂度：O(n)，其中 n 为数组的大小。只需要访问 nums 数组一遍，共 n 个位置。

空间复杂度：O(1)，不需要额外的空间开销。


### 自己之前的错误解答：

对比正确的，少了 if(i <= rightMax) ，少了这句话代表什么


[0,2,3]    
实际：false   
去掉判断： true  

你后面的跳跃，必须在前一个跳跃到达的最远距离之前

在 [0,2,3] 中，如果去了判断，第二次还能蹦，加了判断，是蹦不了的

所以rightMax， 不是，每一个点，能跳到的最大位置，Math.max(i + nums[i], rightMax)

首先你得判断，能不能跳到这个点

```javascript
var canJump = function(nums) {
    let n = nums.length;
    let rightMax = 0;
    for(let i = 0 ; i < n - 1; i++ ) {
        rightMax = Math.max(i + nums[i], rightMax);
        if(rightMax >= n -1) {
            return true
        }
    }
    return false
};
```
### 代码

```javascript
/**
 * @param {number[]} nums
 * @return {boolean}
 */
var canJump = function(nums) {
    let n = nums.length;
    let rightMax = 0;
    for(let i = 0 ; i < n; i++ ) {
        if(i <= rightMax) {
            rightMax = Math.max(i + nums[i], rightMax);
            if(rightMax >= n -1) {
                return true
            }
        }
    }
    return false
};
```


### 跳跃游戏 II

给定一个非负整数数组，你最初位于数组的第一个位置。

数组中的每个元素代表你在该位置可以跳跃的最大长度。

你的目标是使用最少的跳跃次数到达数组的最后一个位置。

示例:

	输入: [2,3,1,1,4]
	输出: 2
解释: 跳到最后一个位置的最小跳跃数是 2。
     从下标为 0 跳到下标为 1 的位置，跳 1 步，然后跳 3 步到达数组的最后一个位置。

### 3. 买卖股票的最佳时机


给定一个数组，它的第 i 个元素是一支给定股票第 i 天的价格。

如果你最多只允许完成一笔交易（即买入和卖出一支股票一次），设计一个算法来计算你所能获取的最大利润。

注意：你不能在买入股票前卖出股票。

 
示例 1:

	输入: [7,1,5,3,6,4]
	输出: 5
解释: 在第 2 天（股票价格 = 1）的时候买入，在第 5 天（股票价格 = 6）的时候卖出，最大利润 = 6-1 = 5 。
     注意利润不能是 7-1 = 6, 因为卖出价格需要大于买入价格；同时，你不能在买入前卖出股票。
示例 2:

	输入: [7,6,4,3,1]
	输出: 0
解释: 在这种情况下, 没有交易完成, 所以最大利润为 0。


### 解题思路

顺序遍历，记录遇到过的最小值
获取 Math.max(前股票数 - 最小值)
时间复杂度：O(n)，只需要遍历一次。
空间复杂度：O(1)，只使用了常数个变量。

### 代码

```javascript
/**
 * @param {number[]} prices
 * @return {number}
 */
var maxProfit = function(prices) {
    let min_price = Infinity;
    let max_profit = 0;
    for(let i = 0; i < prices.length; i++) {
        if(prices[i] < min_price) {
            min_price = prices[i]
        }
        if(prices[i] - min_price  > max_profit) {
            max_profit = prices[i] - min_price
        }
    }
    return max_profit;
};
```
